# 주문 패턴

카카오톡 발주방에서 관찰된 주문 유형 비교 정리.

## 매출처별 주문 스타일 비교

| 항목 | 아리식당 | 가영식당 | 태종대요양병원 | 파크사이드재활의학병원 | 윌리스요양병원 |
|------|---------|---------|--------------|---------------------|--------------|
| **포맷** | 품목수량 **붙여쓰기** (최근 띄어쓰기 추가) | 품목수량 **띄어쓰기** 구분 | **엑셀 파일(.xlsx)** 발주 | **엑셀 파일(.xlsx)** 발주 (`하나푸드발주서.xlsx` 고정) | **엑셀 파일(.xlsx)** 발주 (일별 파일) |
| **수량 단위** | 개, 박스, k(kg), 망, 판, 대 | 키로, 봉지, 박스, 통, 단, 판, 만원, 근 | 봉, 박스, kg, 팩, 판, 망, 통, 말, 손 | kg, 박스, 봉, 개, 망, 팩 | 봉, 박스, kg, 개, 통, 판, 망, 팩 |
| **추가주문** | `추가요`, `추가해주세요`, `품목명만` | 별도 메시지로 추가 | `추가해주세요`, `추가발주합니다` | **우측 추가란** (열13~15) 사용 | `입고부탁드립니다`, `추가입니다`, `같이 입고` |
| **취소** | `품목취소해주세요` | (직접 취소 메시지) | `취소해주세요` (일반 텍스트) | — (엑셀 내 취소) | `취소입니다`, `취소` |
| **정정** | 잘못된 품목명 정정 | 잘못된 발주 수량 정정 | 엑셀 자체 수정 or 텍스트 정정 | 엑셀 자체 수정 | 취소 후 재추가 |
| **평균 주문량** | 중간 규모 (일일 10-20항목) | 중간 규모 (일일 10-20항목) | 대규모 (엑셀 30~100+행) | 중대규모 (엑셀 35~70행) | 대규모 (일별 엑셀) |
| **파일명** | — | — | `M월 D D D .xlsx` | `하나푸드발주서.xlsx` (고정) | `요일M-D.xlsx` → `M-D 요일.xlsx` |
| **배송일 지정** | `고기는월요일주세요` | 날짜 직접 명시 | 엑셀 시트별 | A2 셀 날짜 (D-1 패턴) | `M/D(요일)에 입고해주세요` |
| **가격 문의** | 거의 없음 | 가끔 | **매우 빈번** | — | 가끔 |

## 1. 텍스트 주문 (기본형)

### 아리식당 스타일 — 붙여쓰기
품목과 수량을 공백 없이 붙여서 연속 전송:

```
램33ㅡ1개두부1무1청양고추1국거리3마요네즈2부지갱이6간마늘2양상추1적채2콩나물5고비1양파1
```

**장점:** 한 줄로 많은 품목 전송 가능
**단점:** 품목명과 수량 구분이 어려움

### 가영식당 스타일 — 띄어쓰기
품목과 수량을 공백으로 구분:

```
산취나물2키로 꽃소금1포 줄줄이비엔나3봉지 깻잎지2키로
```

## 2. 추가 주문
첫 주문 후 별도 메시지로 품목 추가:

**키워드 방식 (아리식당):**
```
설탕1추가요
강정2추가요
돈가스1박스추가요
닭도리2추가줌해주세요
```

**단순 추가 (가영식당):**
```
감자1박스
가자미3박스
대파2단 냉동고등어토막1박스
```

## 3. 주문 취소
```
돈가스취소해주세요
```

## 4. 정육(고기) 주문 특징
정육(돼지고기)은 용도별로 나누어 주문:

```
정육20
국거리8
찌게용8
```

- `정육` = 일반 돼지고기 (숫자는 kg)
- `국거리` = 국/찌개용 고기
- `찌게용` = 찌개용 고기
- `불고기용` = 불고기용 고기
- `제육` = 제육볶음용 고기
- `카레용고기` = 카레용 고기
- 배송일 지정 가능: `고기는월요일주세요`, `내일보내주세요`

## 5. 사진 주문
- 사진(촬영한 발주서/메모)으로 대체 가능
- 주문자는 사진을, 호스트는 확인 메시지 전송

## 6. 수량 단위 분류
- **개수:** 개, 대, 판(두부), 통(세제/액체)
- **중량:** kg/k(킬로), 키로, g, 근(고춧가루)
- **포장:** 박스, 봉지/봉, 망(그물), 단(배추/파 묶음)
- **금액:** 만원 (예: 콩나물1만원 = 만원어치)
- **기타:** 팩, 병

## 7. 엑셀 파일 발주

일부 발주처는 텍스트 메시지 대신 **엑셀 파일(.xlsx)** 을 첨부하여 발주합니다.

### 태종대요양병원 스타일 — 여러 날짜를 한 파일에

```
[정원주] 파일: 4월 21 23 25  .xlsx
[정원주] 4월 23일 수요일 입고분 발주입니다
```

- 파일명: `M월 D D D .xlsx` (입고일 나열)
- 주기: 월/수/금 정기 입고 (가끔 토요일)
- 시트: 입고일별로 정리

### 윌리스요양병원 스타일 — 일별 파일

```
[박운재] 파일: 수4-23.xlsx   → 초기: 요일약자M-D.xlsx
[박운재] 파일: 1-5 월.xlsx    → 후기: M-D 요일.xlsx
```

- **매일** 별도 파일 전송 (입고일마다 1개)
- 파일명 변화: `수4-23` → `1-5 월.xlsx` (M-D 요일)
- 응답: `특이품목 없습니다~` (정형화된 확인)

### 추가/취소 (텍스트)
```
새우살 2봉 추가해주세요
무 1박스 취소해주세요
```
- `추가해주세요` / `추가발주합니다` / `취소해주세요`
- 단위: 봉, 박스, kg, 팩, 판, 망, 통, 말, 손

## 7. 발주가 아닌 메시지 (행정 안내·계산·확인 요청)

일부 메시지는 발주·취소·수정 파이프라인에 들어와서는 안 됩니다. 코드에서는
`is_excluded_order_message()` 가 이를 1차로 거르고, AI 파서가 추가로 분류합니다.
봇이 false positive 로 발주 프리뷰를 만들면 매출처 신뢰도가 떨어지고, 도 인호가
수동으로 정리해야 합니다.

### 7-1. 강한 행정 키워드 (false positive 위험)

다음 키워드가 메시지에 포함되어 있고, **단위(박스/키로/봉/통 등) 동반 발주 신호가 없으면**
발주 파이프라인에서 제외합니다.

| 키워드 | 의미 | 예시 |
|--------|------|------|
| `마감` | 일계 마감 금액 / 발주 마감 시간 | `마감 18,093,986 그대로 올려주시고 명세표 보내주세요` |
| `명세표` / `명세서` / `거래명세서` | 거래명세서 요청 | `어제자 명세표 PDF 보내주세요` |
| `세금계산서` / `견적서` | 세금/견적 문서 요청 | `이번달 세금계산서 부탁드립니다` |
| `입금` / `송금` / `송장` | 입금 확인 / 송금 알림 | `입금 500,000원 확인 부탁` |
| `정산` / `청구` / `대금` / `잔금` / `미수금` / `미수` | 정산·청구 | `이번달 정산 부탁드립니다` |
| `선불` / `착불` | 결제 방식 | `착불로 보내주세요` |

### 7-2. 혼합 케이스 (단위 동반 + 행정 키워드)

**단위 동반 발주 신호가 같이 있으면 발주 파이프라인으로 진입**합니다. 발주 부분만
파서가 추출하여 처리하고, 행정 부분은 무시됩니다.

- `배추 1박스 추가, 마감 18,000원 입금 확인` → 발주 진입 ✓ (`1박스` 단위 동반)
- `오이 2키로 추가 마감 18,000원 입금 확인` → 발주 진입 ✓ (`2키로` 단위 동반)

### 7-3. 변경 이력

- **2026-06-04**: 강한 행정 키워드 + (단위 동반) 발주 신호 부재 시 발주에서 제외하는
  로직 추가. 영향: 큰힘병원에서 `마감 18,093,986 그대로 올려주시고 명세표 보내주세요`
  메시지가 발주 프리뷰로 잘못 진입하던 false positive 수정. (`is_excluded_order_message`
  의 AND-NOT 가드 강화 — `hana_bot/order_dates.py::_STRONG_ADMIN_KEYWORDS_RE`,
  `has_strong_order_signal()`)
