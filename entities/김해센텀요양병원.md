### 중요 파싱 규칙
1. **시트명 = 발주일** (예: `5월25` 시트 = 2026-05-25). A3 `일자:`와 항상 일치.
2. **수량(qty)이 빈칸(' ' 또는 '')인 행은 실제 발주가 아님** → 제외하거나 qty=1로 처리
   - 단, **qty가 비어있으면 매칭 대상에서도 제외됨** (실제 발주 수량이 없으므로 주문이 아닌 항목)
   - 예: `가이피라`(qty=''), `버터해드`(qty=''), `양상치`(qty=''), `알배추`(qty=''),
     `라디치오`(qty=''), `적채`(qty=''), `완숙토마토`(qty='') — 모두 비발주 항목
3. **`규격` 컬럼의 값은 품명에 병합되어 처리됨** — Gemini가 자동으로 품명에 `(규격)` 형태로 포함시킴
   - 예: 품명=`바나나`, 규격=`초록` → `바나나(초록)` 으로 매칭 전처리
   - 예: 품명=`돈전지`, 규격=`손다짐` → `돈전지(손다짐)`
   - 예: 품명=`갈치`, 규격=`구이` → `갈치(구이)`
   - 예: 품명=`소양지`, 규격=`국` → `소양지(국)`
   - 즉 DB 매칭 시 `품명1 LIKE '%바나나(초록)%'` 형태로 검색됨
4. **단위 `K` = kg** (일부 행에서 `K`로 표기)
5. **헤더 아래 109개 데이터 행 중 실제 발주가 있는 행만 포함** (보통 10~40행)
6. **여러 날짜가 하나의 파일에 포함**되므로 시트별로 각각 파싱
7. **포장상태(F열)에 절단 치수 정보(`N*N*Ncm` 등)가 있으면 품명에 병합**
   - 예: 품명=`소양지`, 규격=`국`, 포장상태=`2*2*0.5cm` → `소양지(2*2*0.5cm)` 로 처리 (규격보다 포장상태 우선)
   - 규격(C열)의 `국` 대신 포장상태(F열)의 치수 정보를 사용하는 특수 케이스
   - DB 매칭 시 `품명1 LIKE '%소양지(2*2*0.5cm)%'` 로 검색
8. **포장단위 분할 (F/G열 중량값 기준)** — **중요!**
   - 한 품목 행의 **qty가 2개 이상의 포장단위로 구성**된 경우가 있음
   - **F열(포장상태) 또는 G열**에 `1K`, `0.3K`, `0.5kg`, `1kg` 등의 개별 포장단위 값이 있고,
     그 값들의 **합계가 qty와 일치**하면 → **해당 항목을 개별 행으로 분할**
   - 예: 품명=`돈전지`, 규격=`손다짐`, 수량=`1.3`, 단위=`K`, 포장상태=`1K`, G열=`0.3K`
     → `돈전지(손다짐)` 1K + `돈전지(손다짐)` 0.3K (2개 항목으로 분할)
   - 분할된 각 항목은 개별 DB 행으로 저장됨
   - 파서(Gemini)가 이 규칙에 따라 자동 분할 처리

### 7️⃣ 텍스트 발주: "ㅡ" 접미사 규칙
가영식당과 달리 김해센텀은 텍스트 발주 시 **`-`(하이픈) 대신 `ㅡ`(한글 채움 문자)**를 사용합니다.
```
대파ㅡ2단     → 대파 2단
양파 ㅡ 1망   → 양파 1망
적양파ㅡ1망   → 적양파 1망
```
- `ㅡ`는 품목명과 수량 사이의 구분자 역할
- 파서가 자동으로 `ㅡ`를 제거하고 품목명만 추출
