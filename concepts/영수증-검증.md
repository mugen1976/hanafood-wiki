# 영수증 검증 (단가·금액 비교)

매입처(공급자)가 발송방으로 보내는 영수증/거래명세서 사진을 자동으로 인식하여 OCR, DB 발주 내역과 비교, 단가와 금액을 DB에 저장하는 시스템.

**관련 파일:** `receipt_processor.py`, `receipt_comparator.py`, `handlers.py`, `db.py`

## 전체 흐름

```
매입처 발송방 영수증 사진 수신 (type 16386)
     │
     ▼
① detect_receipt_image() — 사진 메시지 감지 (receipt_processor.py)
     │
     ▼
② find_binding_by_room_id() — 매입처 식별 (delivery_registry)
     │
     ▼
③ download_receipt_image() — 카카오 CDN 이미지 다운로드
     │
     ▼
④ ReceiptProcessor.ocr() — Gemini Vision OCR (receipt_processor.py)
     ├─ 인쇄체·손글씨 모두 처리
     └─ 출력: ReceiptData(items[], total_amount, receipt_date)
     │
     ▼
⑤ compare_order_with_receipt() — 발주 DB와 비교 (receipt_comparator.py)
     ├─ 품명2(원문) / 수량 / 단위 기준 fuzzy 매칭
     ├─ 금액 검증: DB수량 × 영수증단가 vs 영수증금액
     └─ 출력: ComparisonResult(매칭·불일치·누락 항목)
     │
     ▼
⑥ update_order_prices() — 단가·금액 DB 저장 (db.py)
     ├─ UPDATE 발주 SET 단가=?, 금액=? WHERE 번호=?
     └─ 매칭된 항목만 저장
     │
     ▼
⑦ format_comparison_report() → 관리방 알림
     ├─ 정상 매칭 항목
     ├─ 수량/금액 불일치 항목
     ├─ 영수증에만 있는 항목
     └─ 총액 비교
```

## 트리거 조건

다음 조건을 모두 만족하면 자동 실행:
- 메시지가 **사진 타입**(16386)이고
- 발송한 방이 `delivery_registry`에 등록된 **매입처 발송방**이며
- attachment에 **이미지 URL**이 존재하는 경우

## ReceiptComparator (receipt_comparator.py)

OCR 추출 데이터와 발주 DB 내역을 비교하여 금액 오류를 탐지.

### 매칭 알고리즘

1. **품명2 정규화** (`_normalize_name`): 괄호 내용 제거, 소문자 변환, 공백 정리
2. **유사도 점수 산정** (`_score_name_match`):
   - 정확 일치 = 1.0
   - 부분 포함 = 0.9
   - 문자 공통 비율 (common/max(len))
3. **매칭 임계값:** 0.3 이상인 최적 항목 선택
4. **금액 검증:** `DB수량 × 영수증단가` vs `영수증금액` 차이 계산
5. **상태 분류:**
   - `matched` — 정상
   - `qty_mismatch` — 수량 차이
   - `price_mismatch` — 금액 차이
   - `missing_in_receipt` — DB에는 있지만 영수증에 없음

### 비교 결과 구조

```python
@dataclass
class ComparisonItem:
    ord_no: int                    # 발주.번호 (UPDATE용)
    db_item_name: str              # DB 품명2
    db_qty: float                  # DB 수량
    db_unit: str                   # DB 단위
    receipt_item: ReceiptItem | None    # 매칭된 영수증 항목
    match_status: str              # matched / qty_mismatch / price_mismatch / ...
    expected_amount: float | None  # DB수량 × 영수증단가
    unit_price: int | None         # 영수증 단가
    receipt_amount: int | None     # 영수증 금액
    discrepancy: float | None      # 차액 (receipt_amount - expected_amount)
    qty_diff: float                # 영수증 수량 - DB 수량

@dataclass
class ComparisonResult:
    purchase_name: str
    order_date: date
    items: list[ComparisonItem]
    db_total_count: int
    receipt_total_amount: int | None
    db_expected_total: float
    total_discrepancy: float
    has_errors: bool
    unmatchable_receipt_items: list[ReceiptItem]  # DB에서 찾을 수 없는 영수증 항목
```

## DB 저장

`발주` 테이블에 추가된 컬럼:

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `단가` | int | 영수증에서 추출한 단가 |
| `금액` | int | 영수증에서 추출한 금액 (단가×수량) |

**저장 시점:** OCR → DB 매칭 완료 후, 매칭된 행에 대해 `update_order_prices(번호, 단가, 금액)` 실행.

```python
# db.py
def update_order_prices(self, ord_no: int, unit_price: int, amount: int) -> bool:
    """발주 단가·금액 업데이트 (영수증 매칭 결과)."""
    with self.cursor() as cur:
        cur.execute(
            "UPDATE 발주 SET 단가 = ?, 금액 = ? WHERE 번호 = ?",
            unit_price, amount, ord_no,
        )
        return cur.rowcount > 0
```

## 핸들러 통합 (handlers.py)

```python
def _handle_receipt_photo(self, ctx) -> bool:
    """매입처 발송방의 영수증 사진을 감지·OCR·DB 비교."""
    # 1. 사진 메시지 감지
    img_info = detect_receipt_image(ctx)
    if img_info is None: return False

    # 2. 매입처 발송방 확인
    binding = self.delivery_registry.find_binding_by_room_id(img_info.room_id)
    if binding is None: return False

    # 3. 이미지 다운로드
    local_path = download_receipt_image(img_info)
    # 4. Gemini Vision OCR
    receipt = ReceiptProcessor(self.settings.gemini_key).ocr(local_path)
    # 5. 발주 DB 조회
    db_orders = self.db.fetch_orders_by_purchase_and_date(purchase_name, order_date)
    # 6. 비교
    result = compare_order_with_receipt(db_orders, receipt, ...)
    # 7. 단가/금액 DB 저장
    for item in result.items:
        if item.receipt_item and item.unit_price is not None:
            self.db.update_order_prices(item.ord_no, item.unit_price, item.receipt_amount)
    # 8. 결과 알림
    self._reply_admin_room(ctx, format_comparison_report(result))
    return True
```

**호출 시점:** `on_chat()`에서 엑셀 첨부 검사 직후, 발주방/관리방 라우팅 이전에 실행.
