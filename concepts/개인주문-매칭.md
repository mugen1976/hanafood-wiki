# 개인주문 매칭 규칙

## 개요
발주 메시지에 `개인주문` 키워드가 포함된 경우, 해당 매출처의 `(개인주문)` 서브매출처의 품명 테이블도 함께 검색해야 한다.

## 적용 흐름

1. 메시지 본문에 `개인주문` 문자열 포함 여부 확인
2. 포함 시, `{sales_name}(개인주문)` 을 `alt_sales_names` 로 추가
3. `Matcher.resolve()` 에서 기본 sales_name 으로 검색 → 후보 0건인 항목에 대해 alt_sales_names 로 재검색
4. (개인주문) 매출처에서 찾은 품목은 정상 매칭 처리

## 예시

| 메시지 | room 매출처 | 추가 검색 대상 |
|--------|-------------|---------------|
| `개인주문으로 양념깻잎지0.5킬로, 무말랭이지0.5킬로 부탁드립니다` | 정다운요양병원 | 정다운요양병원(개인주문) |
| `[개인주문] 된장고추지 0.2kg` | 윌리스요양병원 | 윌리스요양병원(개인주문) |

## 관련 코드

- `hana_bot/matcher.py` → `Matcher.resolve()` 의 `alt_sales_names` 파라미터
- `hana_bot/order_room_handler.py` → `resolve_order_content()` 에서 `개인주문` 감지

## 참고
- `(개인주문)` 매출처는 DB `매출처` 테이블에 별도 레코드로 존재 (번호=28 등)
- `(개인주문)` 품명 테이블은 메인 매출처와 품목 구성이 다를 수 있음 (예: `무말랭이지`는 개인주문에만 등록)
