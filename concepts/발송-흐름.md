# 발송 흐름

발주방의 발송 확인부터 결과까지의 전체 프로세스.

## 전체 흐름

```
① 봇이 발주 데이터 취합
   ↓
② 발주 미리보기 생성 (이미지 + 텍스트)
   ↓
③ 사용자 !발송확인 N 명령어 입력
   ↓
④ 봇이 각 공급처별로 발송 실행
   ↓
⑤ 발송 결과 리포트 출력
```

## 상세 단계

### ① 데이터 취합
- 봇이 각 매입처(공급처)에서 들어온 주문을 취합
- 총 건수 및 매입처 구분 자동 분류

### ② 미리보기 생성
- **발송 이미지:** 발주 내역 이미지 생성
- **발주일 지정:** 예: `발주일: 2026-05-18`
- **매입처 분류:**
  - `[등록됨 · 전체 발송 대상]` — 기본 발송 대상
  - `[추가발송만 등록 · 전체 발송 제외]` — 선택적 발송
  - `[발송방 미등록 · 발송 제외]` — 발송 불가
- **모드 표시:** 기본, 차량, 기본차량

### ③ 발송 확인
사용자가 `!발송확인` 명령어로 승인
- 전체 발송 또는 번호 지정 발송 가능

### ④ 발송 실행
- 선택된 매입처로 각각 발송
- `스킵(선택 발송 제외)` — 선택 안 한 매입처는 스킵
- `이미지 1장 발송 (확인됨)` — 이미지 발송 완료

### ⑤ 결과 리포트
- 매입처별 발송 상태 표시
- 건수 및 발송 방식 표시
